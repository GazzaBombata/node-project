API Documentation

User Endpoints

  • POST /users: Creates a new user. The request body should contain the user details.
  • GET /users/:id: Retrieves a user by ID.
  • PUT /users/:id: Updates a user by ID. The request body should contain the new user details.
  • DELETE /users/:id: Deletes a user by ID.

Post Endpoints

  • POST /posts: Creates a new post. The request body should contain the post details.
  • GET /posts/:id: Retrieves a post by ID.
  • PUT /posts/:id: Updates a post by ID. The request body should contain the new post details.
  • DELETE /posts/:id: Deletes a post by ID.
  • GET /posts: Retrieves all posts with interactions. The request can include query parameters for filtering.

Interaction Endpoints

  • POST /interactions: Creates a new interaction. The request body should contain the interaction details.
  • GET /interactions/:id: Retrieves an interaction by ID.
  • PUT /interactions/:id: Updates an interaction by ID. The request body should contain the new interaction details.
  • DELETE /interactions/:id: Deletes an interaction by ID.
